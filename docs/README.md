## Scope Rule

Keep only active, decision-grade documentation here.
Historical plans/audits/checklists should not remain in the living set.

## Style Guide

All documents in this repository should follow:

1. English language.
2. Clear scope section.
3. Operationally useful content (decisions, rules, commands, references).
4. No duplicated roadmap/checklist fragments across files.
